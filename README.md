**ABSTRACT**

This documentation presents the development of the AERMOD Open software
to support and automate the generation of the input files of the AERMOD
atmospheric dispersion modeling system, thus contributing to
professionals and/or researchers in the field. The software should
automate the configuration of your input files for each AERMAP, AERMET,
and AERMOD main processor processing step. The professional will have a
user-friendly interface making it easy to perform on the dispersion
model effectively. The system was implemented in .Net C# through the
Visual Studio Community 2022 development platform, using the MySQL
5.6.20 Relational Database and the methodology used was Object-Oriented
Analysis.

**GOALS**

The main objective of the software is to automate the generation of
input files for the AERMAP, AERMET and AERMOD processing steps of the
AERMOD atmospheric dispersion modeling system. Through a user-friendly
interface, the software allows you to manage emitting sources,
topographic, meteorological and radiosonde files. With this, the system
will assist environmental professionals, ensuring greater efficiency in
the simulation periods for each pollutant configured.

**3.1 AERMAP process**

3.1.1 File Registration (TIF)

3.1.1.1 The software allows the user to import topographic files with
extension (TIF);

3.1.1.2 The system allows the deletion of files according to the user's
needs;

3.1.1.3 The system can export selected files.

3.1.2 Emission sources

3.1.2.1 If the emitting source is not registered in the system, the
system has a register of emitting sources with specific information and
appropriate coordinates;

3.1.2.2 The system may insert an emitting source if it is not registered
in the system;

3.1.2.3 The system can change the data of the source if it is
registered;

3.1.2.4 The system may exclude the registered issuing source and is not
linked to another table in the system;

3.1.2.5 The system can perform queries to make possible changes.

3.1.3 Domain and Modeling Grid

3.1.3.1 The system has a domain registration and modeling grid if it is
not registered in the system, requesting specific data from the study
area;

3.1.3.2 The system can change the registered record if necessary;

3.1.3.3 The system can delete the selected record through the existing
query in the register;

3.1.3.4 The system can perform queries to make possible changes.

3.1.4 Implementation

3.1.4.1 The system allows the execution of the AERMAP preprocessor by
generating output files such as: AERMAP. INP, AERMAP. OUT, AERMAP. ROU
and AERMAP. AM;

3.1.4.2 The system provides option to export output files generated by
processing.

**3.2 AERMET process**

3.2.1  File Registration (FSL)

3.2.1.1 The software allows the user to import radio sounding files with
extension (FSL);

3.2.1.2 The system allows the deletion of files according to the user's
needs;

3.2.1.3 The system can export selected files.

3.2.2  File Registration (SAM)

3.2.2.1 The software allows the user to import hourly surface
meteorological files with extension (CSV) generated by the National
Institute of Meteorology INMET;

3.2.2.2 The system allows the deletion of files as needed by the user;

3.2.2.3 The system can export selected files;

3.2.2.4 The system converts files (CSV) to (SAM) according to parameters
defined in the registration by the user.

3.2.3  Registration of definitions

3.2.3.1 If there is no definition of surface characteristics registered
in the system, it has a register of definitions with information on
specific characteristics and due period for simulations;

3.2.3.2 The system can enter a definition if it is not registered in the
system;

3.2.3.3 The system can change the definition data if it is registered;

3.2.3.4 The system may delete the definition registered and it is not
linked to another table in the system;

3.2.3.5 The system can perform queries to make possible changes.

3.2.4  Execution

3.2.4.1 The system allows the execution of the AERMET preprocessor by
generating output files such as AERMET_1.INP, AERMET_2.INP,
AERMET_3.INP, AERMET. PFL and AERMET. SFC;

3.2.4.2 The system provides option to export output files generated by
processing.

**3.3  AERMOD Process**

3.3.1  Registration of definitions

3.3.1.1 If there is no definition of time averages, pollutants and
output files registered in the system, it has a register of definitions
with information on time averages, hourly, the entire simulation period,
pollutants and output files;

3.3.1.2 The system may enter a definition if it is not registered in the
system;

3.3.1.3 The system can change the definition data if it is registered;

3.3.1.4 The system may delete the registered definition;

3.3.1.5 The system can perform queries to make possible changes.

3.3.2  Parameter registration

3.3.2.1 If there are no parameters defined for the emitting sources
registered in the system, it has a register of parameters with specific
information for each type of emitting source, being point, area, polygon
area or circle area;

3.3.2.2 The system can change the data of the parameter of the emitting
source if it is registered;

3.3.2.3 The system can perform queries to make possible changes.

3.3.3  Execution

3.3.3.1 The system makes the execution of the AERMOD main processor
available by generating output files as AERMOD. INP, AERMOD. OUT,
ERRORS. OUT and AERMOD. PLT;

3.2.4.2 The system provides option to export output files generated by
processing.

**3.4  Map**

3.4.1  AERMAP Map

3.4.1.1 The system allows the visualization of the map of the study
area, containing the domain, modeling grid and emission sources
registered in the system.

3.4.2  AERMOD Map

3.4.2.1 The system allows the visualization of the contour map with the
maximum average concentrations and their coordinates of each pollutant
and file generated by the main processor AERMOD.

**3.5  Backup**

3.5.1  Make a copy

3.5.1.1 The system allows you to back up the structure, rows or records
of the tables.

3.5.2 Restore copy

3.5.2.1 The system can restore a backup copy if there is one created by
the backup module.

**4.1  System Overview**

The AERMOD modeling system is a Gaussian atmospheric dispersion
regulatory model indicated by the United States Environmental Protection
Agency US EPA. However, preparing the files to feed the modeling system
is not a trivial task, prone to errors and that requires great attention
on the part of professionals. In this sense, this free system aims to
automate the generation of input files for the AERMAP, AERMET and the
main AERMOD processor of the AERMOD modeling system, thus contributing
to professionals and/or researchers in the area. Through a user-friendly
interface, the system will allow the professional to configure their
input files for each processing step, facilitating the execution of the
dispersion model effectively. The results of the system can help
universities by reducing costs with the use of proprietary software and
encouraging new studies on improvements in air quality.

The proposed system has a management of the AERMAP preprocessor, AERMET
and AERMOD main processor presenting the following characteristics:

4.2.3.5 The system must allow changes to registered records if

files as AERMAP. INP, AERMAP. OUT, AERMAP. ROU and AERMAP. AM;

4.2.4.2 The system must issue an error message if there is any missing

4.2.4.3 The system must register in the database the code, file (INP),

4.3.1.1 The system must allow the import of radiosonde files with

4.3.3.3 The system must issue an error message if there is duplication

4.3.3.4 The system must issue an error message if any field is not

4.3.3.5 The system must allow changes to registered records if

4.3.4.2 The system must issue an error message in the execution, if

4.3.4.3 The system provides option to export all output files generated

**4.4  AERMOD Functional Requirements**

4.4.1  Registration of definitions

4.4.1.1 The system shall record time averages using the code, hourly
average, period average, pollutant, unit of measurement, rectangle and
maximum value.

4.4.1.2 The system must record output parameters such as code, sequence,
type of output, hourly average, period average, air quality standard,
receiver criterion, maximum value, description and file;

4.4.1.3 The system must issue an error message if there is duplication
of the data compared to those recorded in the database, both in the
insertion and alteration;

4.4.1.4 The system must issue an error message if any mandatory field is
null/empty or not filled in correctly;

4.4.1.5 The system may change the data of time averages or output
parameters if they are registered;

4.4.1.6 The system can delete the selected record through the existing
query in the register through a confirmation message or not for
deletion;

4.4.1.7 The system can perform queries to make possible changes or
definition of the pollutant for simulations.

4.4.2  Parameter registration

4.4.2.1 The system shall record parameters of the point-type emitting
sources by means of the source code, emission rate, stack height, outlet
temperature, outlet velocity and stack diameter;

4.4.2.2 The system shall record parameters of the area-type emitting
sources through the source code, emission rate, launch height, X and Y
coordinates, angle and vertical dimension;

4.4.2.3 The system shall record parameters of the emitting sources of
the polygon area type through the source code, emission rate, launch
height, number of vertices, vertical dimension and sequence of X and Y
coordinates of the polygon;

4.4.2.4 The system shall record parameters of the emitting sources of
the circle area type through the source code, emission rate, launch
height, radius, number of vertices and vertical dimension;

4.4.2.5 The system must issue an error message if any mandatory field is
null/empty or filled incorrectly;

4.4.2.6 The system can change the parameters of the selected emitting
source if necessary.

4.4.3  Execution

4.4.3.1 The system must run the AERMOD main processor generating output
files as AERMOD. INP, AERMOD. OUT, ERRORS. OUT and AERMOD. PLT;

4.4.3.2 The system must issue an error message in the execution, if
there is any missing or incorrect data in the registration of
definitions or parameters of AERMOD or in the AERMAP or AERMET
processes;

4.3.4.3 The system provides option to export all output files generated
by processing;

4.3.4.4 The system must issue a confirmation message after exporting the
desired file.

**4.5  Functional requirements of the map**

4.5.1  AERMAP Map

**12. IMPLEMENTATION**

The software, source code, database, and installation guide are
available on the platform *web*
[www.aermod.infinityfreeapp.com](http://www.aermod.infinityfreeapp.com).
Professionals, researchers or students in the area can use the software
for simulations, atmospheric dispersion modeling and, if necessary,
future implementations or improvements. For more information or
questions, please contact us via email
<aermod.infinityfreeapp@gmail.com> or <borges2@hotmail.com>.

If there is a version update, the user will be notified by the software
about the new version, allowing them to abort or complete the
installation process. The update can also be done through the platform
*web* Doing the *download* of the installer and carrying out its
execution.
